## Overview

`MatrixClient` is the main class in matrix-js-sdk. It wraps the Matrix Client-Server API and provides a high-level interface for authentication, room operations, messaging, media handling, and real-time sync. It extends `TypedEventEmitter`, so you can listen for events using `.on()`, `.once()`, and `.off()`.

You should not instantiate `MatrixClient` directly. Use [`createClient()`](/api/create-client) instead.

```typescript
import { createClient } from "matrix-js-sdk";

const client = createClient({
  baseUrl: "https://matrix.org",
  accessToken: "syt_...",
  userId: "@alice:matrix.org",
});
```

## Client Lifecycle

### `startClient(opts?)`

Start syncing with the homeserver. Must be called before most other methods will return live data.

```typescript
public async startClient(opts?: IStartClientOpts): Promise<void>
```

<ParamField body="opts" type="IStartClientOpts" optional>
  Configuration for the sync loop.

  <Expandable title="IStartClientOpts properties">
    <ParamField body="initialSyncLimit" type="number" optional>
      The `limit=` applied to the initial sync request. Default: `8`.
    </ParamField>
    <ParamField body="includeArchivedRooms" type="boolean" optional>
      Include archived rooms in the initial sync. Default: `false`.
    </ParamField>
    <ParamField body="resolveInvitesToProfiles" type="boolean" optional>
      Fetch profile info for unknown invite senders. Default: `false`.
    </ParamField>
    <ParamField body="pendingEventOrdering" type="PendingEventOrdering" optional>
      Where pending (unsent) messages appear in a room timeline. One of `"chronological"` (default) or `"detached"`.
    </ParamField>
    <ParamField body="pollTimeout" type="number" optional>
      Milliseconds to wait on `/sync`. Default: `30000`.
    </ParamField>
    <ParamField body="filter" type="Filter" optional>
      A `Filter` object to apply to `/sync` calls.
    </ParamField>
    <ParamField body="disablePresence" type="boolean" optional>
      Sync without automatically updating presence. Default: `false`.
    </ParamField>
    <ParamField body="lazyLoadMembers" type="boolean" optional>
      Defer loading room membership events until needed. Default: `false`.
    </ParamField>
    <ParamField body="clientWellKnownPollPeriod" type="number" optional>
      Seconds between polls to `/.well-known/matrix/client`. Default: `undefined` (disabled).
    </ParamField>
    <ParamField body="threadSupport" type="boolean" optional>
      Organise events into threaded conversations when thread relations are found.
    </ParamField>
    <ParamField body="slidingSync" type="SlidingSync" optional>
      **Experimental.** Use a sliding-sync connection instead of the classic `/sync` loop.
    </ParamField>
  </Expandable>
</ParamField>

<ResponseField name="Promise\<void\>" type="Promise">
  Resolves once the sync loop has started.
</ResponseField>

```typescript
await client.startClient({
  initialSyncLimit: 20,
  lazyLoadMembers: true,
});
```

---

### `stopClient()`

Stop syncing and perform a clean shutdown of the client.

```typescript
public stopClient(): void
```

This cancels any in-flight sync requests, clears interval timers, and stops the crypto backend. Safe to call multiple times.

```typescript
client.stopClient();
```

---

### `clearStores(args?)`

Delete all data from the persistent stores used by this client. The client must **not** be running when this is called.

```typescript
public clearStores(
  args: { cryptoDatabasePrefix?: string } = {}
): Promise<void>
```

<ParamField body="args.cryptoDatabasePrefix" type="string" optional>
  The IndexedDB database name prefix to clear. Defaults to `'matrix-js-sdk'`.
</ParamField>

<ResponseField name="Promise\<void\>" type="Promise">
  Resolves when all stores have been cleared.
</ResponseField>

```typescript
client.stopClient();
await client.clearStores();
```

### `joinRoom(roomIdOrAlias, opts?)`

## Messaging

### `sendEvent(roomId, eventType, content, txnId?)`

Send a Matrix timeline event to a room.

```typescript
public sendEvent<K extends keyof TimelineEvents>(
  roomId: string,
  eventType: K,
  content: TimelineEvents[K],
  txnId?: string,
): Promise<ISendEventResponse>

// Thread-scoped variant:
public sendEvent<K extends keyof TimelineEvents>(
  roomId: string,
  threadId: string | null,
  eventType: K,
  content: TimelineEvents[K],
  txnId?: string,
): Promise<ISendEventResponse>
```

<ParamField body="roomId" type="string" required>
  The ID of the room to send the event to.
</ParamField>
<ParamField body="threadId" type="string | null" optional>
  The event ID of the thread root. Pass `null` for the main timeline.
</ParamField>
<ParamField body="eventType" type="string" required>
  The Matrix event type, e.g. `"m.room.message"`.
</ParamField>
<ParamField body="content" type="IContent" required>
  The event content object.
</ParamField>
<ParamField body="txnId" type="string" optional>
  Optional transaction ID to deduplicate repeated attempts.
</ParamField>

<ResponseField name="event_id" type="string">
  The ID assigned to the sent event by the homeserver.
</ResponseField>

```typescript
const { event_id } = await client.sendEvent(
  "!abc123:matrix.org",
  "m.room.message",
  { msgtype: "m.text", body: "Hello!" },
);
```

### `mxcUrlToHttp(mxcUrl, width?, height?, resizeMethod?, ...)`

Convert an MXC URI to an HTTP URL.

```typescript
public mxcUrlToHttp(
  mxcUrl: string,
  width?: number,
  height?: number,
  resizeMethod?: string,
  allowDirectLinks?: boolean,
  allowRedirects?: boolean,
  useAuthentication?: boolean,
): string | null
```

<ParamField body="mxcUrl" type="string" required>
  The `mxc://` URI to convert.
</ParamField>
<ParamField body="width" type="number" optional>
  Desired thumbnail width in pixels.
</ParamField>
<ParamField body="height" type="number" optional>
  Desired thumbnail height in pixels.
</ParamField>
<ParamField body="resizeMethod" type="string" optional>
  Thumbnail resize method: `"crop"` or `"scale"`.
</ParamField>
<ParamField body="allowDirectLinks" type="boolean" optional>
  If `true`, return non-MXC URLs as-is. This may leak information. Default: `false`.
</ParamField>
<ParamField body="allowRedirects" type="boolean" optional>
  If `true`, the caller accepts 307/308 redirects. Default: `false`.
</ParamField>
<ParamField body="useAuthentication" type="boolean" optional>
  If `true`, return an authenticated media URL. Implies `allowRedirects`. Default: `false`.
</ParamField>

<ResponseField name="string | null" type="string | null">
  The HTTP URL, or `null` if the MXC URL is invalid.
</ResponseField>

```typescript
const url = client.mxcUrlToHttp(
  "mxc://matrix.org/abc123",
  96,   // width
  96,   // height
  "crop",
);
```

## Authentication

### `login(loginType, data)`

Post a login request. Updates the client with returned credentials.

```typescript
public login(
  loginType: LoginRequest["type"],
  data: Omit<LoginRequest, "type">,
): Promise<LoginResponse>
```

<Warning>
  This method is deprecated. It mutates the `MatrixClient` instance inconsistently. Prefer calling `loginRequest()` and constructing a new `MatrixClient` with the returned credentials. See [matrix-org/matrix-js-sdk#4502](https://github.com/matrix-org/matrix-js-sdk/issues/4502).
</Warning>

<ParamField body="loginType" type="string" required>

</ParamField>
<ParamField body="data" type="object" required>
  Flow-specific credentials, e.g. `{ user, password }` for `m.login.password`.
</ParamField>
